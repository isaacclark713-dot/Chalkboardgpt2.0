# Call Sheet — Coach Brain v6

Static GitHub Pages version of the Call Sheet app.

## What changed in v6

- Overhauled play-result logging.
- The diagnosis panel is now formation/play/concept specific.
- Motion plays ask how the defense handled motion.
- Pass plays ask for coverage/shell.
- Pass + motion plays ask for both.
- Run plays ask for box/run-fit responses.
- Screens ask whether the screen was open, covered, retraced, or failed by execution.
- Draws ask whether coverage dropped, the box vacated, the DL retraced, or the defense sat on it.
- RPO/read plays ask for apex/conflict-defender behavior.
- Play-action/boot concepts ask how linebackers, safeties, and edges reacted to run action.

The app still stores playbooks and user data locally in each browser. Use the app backup/export tools to move a saved playbook between devices.

## Deploy to GitHub Pages

1. Create a new GitHub repository.
2. Upload all files in this folder.
3. Go to **Settings → Pages**.
4. Select **Deploy from a branch**.
5. Choose `main` and `/ root`.
6. Save.

Your app will be available at your GitHub Pages URL after deployment finishes.
